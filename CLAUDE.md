# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

This is a documentation repository for internal AI resources guides at THG (The Hut Group), an e-commerce company selling beauty products (lookfantastic) and sports nutrition (Myprotein). The documents help employees across all roles understand and use available AI tools.

## Documents

- `ai-resources-guide.md` — Main guide covering all available AI tools, data sensitivity policies, a quick-reference table, role-based use cases with sample prompts, and getting started tips. Audience: all employees.
- `claude-app.md` — Deep-dive into Claude Enterprise: Projects, tips for effective use, and learning resources.
- `claude-code.md` — Deep-dive into Claude Code with THG-specific installation instructions (Unbound Gateway, WSL setup), configuration, and external learning resources.
- `building-agents-for-non-programmers.md` — How to create AI assistants and automated workflows using no-code platforms.

## Conventions

- British English spelling (organisation, summarise, optimised, colour)
- Markdown formatting; escape `$` signs as `\$` to avoid LaTeX rendering issues
- Data sensitivity guidance is framed as warnings/recommendations, not absolute rules
- Tool access details reference ConductorOne (via Okta at thg.okta.com) and Unbound Gateway (ai.thg.dev)
- `[TBD]` markers indicate information that still needs to be confirmed
- Documents may eventually become the basis for presentations
